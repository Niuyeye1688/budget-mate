# BudgetMate 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
BudgetMate — 私人财务管家

### 1.2 产品定位
一款融合 AI 智能审批与规则引擎的个人消费管理软件，以"让预算撑到月底"为核心目标，帮助用户养成理性消费习惯。

### 1.3 产品形态
- **桌面端**：Windows 独立可执行程序（BudgetMate.exe）
- **移动端**：Android APK（基于 Capacitor 混合开发）
- **技术栈**：Python(Flask) + 原生 JavaScript + Capacitor + IndexedDB

### 1.4 核心价值
1. **AI 智能审批**：消费前经 AI 顾问审批，用或不用给出明确建议
2. **预算健康守护**：以"日均可用预算"为核心指标，确保月底不超标
3. **灵活双模式**：AI 模式智能灵活，规则模式严格可控

## 2. 目标用户

| 用户画像 | 特征 | 痛点 |
|---------|------|------|
| 月光族 | 收入固定，月末常超支 | 不知道钱花哪了，缺乏消费约束 |
| 学生党 | 生活费有限，聚餐多 | 冲动消费多，预算管理意识弱 |
| 上班族 | 有记账习惯但难坚持 | 记账繁琐，缺乏实时反馈 |
| 家庭主理人 | 管理家庭开支 | 需要分类管控，大额支出易超 |

## 3. 核心功能模块

### 3.1 消费审批（核心）

#### 3.1.1 单笔审批
- **入口**：首页快速审批
- **流程**：
  1. 输入金额和用途描述
  2. 系统判断消费分类
  3. AI/规则判断是否通过
  4. 显示审批结果和理由
  5. 自动记录到账单

#### 3.1.2 行程审批（批量审批）
- **入口**：行程审批页面
- **场景**：外出就餐、购物、旅行等一次多笔消费
- **流程**：
  1. 添加多项消费（金额+描述）
  2. 系统逐项检测分类
  3. AI 合并判断整体合理性
  4. 返回整体结果和明细

### 3.2 预算管理

#### 3.2.1 月度总预算
- 设置每月可用总预算
- 实时显示剩余预算

#### 3.2.2 分类预算上限
- 餐饮、交通、购物、娱乐、其他五类
- 每类可独立设置月度上限
- 分类上限自动受总预算约束（取较小值）

### 3.3 消费记录
- 审批通过的消费自动入库
- 支持按时间查看记录
- 支持单笔删除
- 支持数据导出

### 3.4 账单统计
- **日报**：当日消费明细和合计
- **周报**：本周消费趋势和分类占比
- **分类统计**：各类消费汇总

### 3.5 AI 配置
- **开关**：启用/禁用 AI 模式
- **API 配置**：base_url、api_key、model
- **饮食偏好**：（扩展字段，如不吃辣、不吃鱼等）

## 4. 审批逻辑架构

### 4.1 双模式设计

```
┌─────────────────────────────────────────────┐
│              消费审批请求                       │
└───────────────────┬───────────────────────────┘
                    │
         ┌──────────▼──────────┐
         │   AI 模式是否启用？   │
         └──────┬──────┬───────┘
                │      │
               是      否
                │      │
      ┌─────────▼      ▼─────────┐
      │    AI 判断（全权）         │   规则引擎判断      │
      │  · 日均预算红线            │   · 总预算检查     │
      │  · 分类合理性              │   · 分类上限检查   │
      │  · 消费习惯                │   · 短时频率限制  │
      │  · 月底目标                │   · 大额提醒      │
      └──────────┬───────────────┘
                   │
            ┌──────▼──────┐
            │  AI 失败？  │
            └──────┬──────┘
                   │
          ┌────────▼────────┐
          │  fallback 到规则引擎  │
          └────────────────────┘
```

### 4.2 AI 审批逻辑（v1.3.9）

#### 核心策略
以"日均可用预算 = 剩余预算 ÷ 剩余天数"为核心红线，灵活判断每笔消费。

#### AI Prompt 关键信息
- 月预算、本月已用、本月剩余
- 当前日期、本月剩余天数
- 日均可用预算（计算值）
- 各分类使用情况

#### AI 判断标准
| 分类 | 判断依据 |
|------|---------|
| 餐饮 | 正常一餐不应明显超过日均预算，聚餐等特殊情况可适当放宽 |
| 交通 | 日常通勤单次不应超过日均预算的2倍，打车非急事拒绝 |
| 购物 | 非必需品优先拒绝，奢侈品直接拒绝 |
| 娱乐 | 每月不超过2-3次，单次不超过日均预算的3倍 |
| 通用 | 明显浪费、冲动消费、可替代方案更便宜的 → 拒绝 |

### 4.3 规则引擎（fallback）

#### 总预算检查
- 总预算 ≤ 0 → 拒绝
- 本月已用 + 本次金额 > 总预算 → 拒绝

#### 分类上限检查
- 分类上限取 min(分类设置值, 月总预算)
- 本月累计 + 本次金额 > 分类上限 → 拒绝

#### 短时频率限制
- **单笔审批**：1小时内同分类累计不超过月预算的 5%
- **行程审批**：1小时内同分类累计不超过月预算的 15%

## 6. 技术架构

### 6.1 桌面端
- **后端**：Flask (Python)
- **前端**：原生 HTML + CSS + JavaScript
- **数据存储**：
  - 开发模式：项目目录 `budget_data.json`
  - 生产模式：`%LOCALAPPDATA%/budget-mate/budget_data.json`
- **打包**：PyInstaller

### 6.2 移动端
- **框架**：Capacitor（混合开发）
- **前端**：与桌面端共用一套 HTML/JS
- **数据存储**：
  - 主存：IndexedDB
  - 降级：localStorage → memory
- **构建**：Android Studio / Gradle

### 6.3 数据持久化三层 fallback
```
IndexedDB → localStorage → memory
```
- **IndexedDB**：首选，支持大容量
- **localStorage**：IndexedDB 不可用时的降级
- **memory**：最后的内存兜底

## 9. 非功能性需求

### 9.1 性能
- AI 审批响应时间 < 3s（取决于 API 服务商）
- 本地规则审批响应时间 < 100ms
- IndexedDB 初始化超时 2s

### 9.2 兼容性
- **桌面端**：Windows 10/11
- **移动端**：Android 8+ (API 26+)

### 9.3 安全
- API Key 本地存储，不上传服务器
- 不收集用户隐私数据
- 数据导出为 JSON 格式
